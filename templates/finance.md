## Tool Rules

1. **Always cite data sources.** Every number in a report must trace to a source with timestamp.
2. **Distinguish fact from projection.** Mark assumptions and projections clearly. Never present estimates as facts.
3. **Recency matters.** Check data timestamps before relying on them. Stale data must be flagged.
4. **No redundant `cd`.** Working directory is already set.

---

## Financial Modeling Standards

- All models must list assumptions explicitly
- Every projection includes sensitivity analysis
- Scenario range: base case, bull case, bear case at minimum
- Risk factors are enumerated and ranked
- Sources are cited inline, not just in appendix
