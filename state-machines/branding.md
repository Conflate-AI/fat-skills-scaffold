# Branding Process State Machine

Every branding/content initiative follows this sequence:

```
STRATEGY ──► PLAN ──► CREATE ──► REVIEW ──► PUBLISH ──► MEASURE ──► ITERATE
  │             │         │         │          │          │           │
  │             │         │         │          │          │           └─ feed data back into STRATEGY
  │             │         │         │          │          └─ track engagement, reach, conversion
  │             │         │         │          └─ post to platform, schedule distribution
  │             │         │         └─ brand voice check, fact check, peer review
  │             │         └─ draft content, follow brand guidelines
  │             └─ content calendar, topic selection, platform mapping
  └─ define brand voice, audience, pillars, goals
```

## Transition guards

| Transition | Guard |
|---|---|
| STRATEGY → PLAN | Brand voice defined. Target audience identified. Content pillars set. |
| PLAN → CREATE | Topics selected and scheduled. Platform assignments made. |
| CREATE → REVIEW | Draft complete. Follows brand voice guidelines. |
| REVIEW → PUBLISH | Brand voice check passed. Facts verified. No tone-deaf moments. |
| PUBLISH → MEASURE | Content is live. Tracking in place. |
| MEASURE → ITERATE | Minimum 1 week of data collected for published content. |

## CREATE detail

1. **Follow brand voice** — use defined tone, vocabulary, and style rules
2. **Platform-native** — adapt content to each platform's norms and constraints
3. **Hook first** — lead with value, not context
4. **Call to action** — every piece has a clear next step for the reader

## REVIEW detail

- **Brand voice check** — does this sound like us? Consistency with guidelines.
- **Fact check** — are all claims accurate and sourced?
- **Sensitivity review** — any tone-deaf or problematic angles?
- **Format check** — correct image sizes, character limits, link formatting.
