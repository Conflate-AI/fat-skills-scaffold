# Go-to-Market Process State Machine

Every GTM initiative follows this sequence:

```
RESEARCH ──► POSITION ──► PLAN ──► LAUNCH ──► MEASURE ──► ITERATE
  │              │           │         │          │           │
  │              │           │         │          │           └─ feed data back into RESEARCH
  │              │           │         │          └─ track KPIs, attribute results to channels
  │              │           │         └─ execute launch plan, coordinate teams
  │              │           └─ detailed launch plan, timelines, owners, budget
  │              └─ define ICP, value prop, messaging, differentiation
  └─ market research, competitive analysis, customer interviews
```

## Transition guards

| Transition | Guard |
|---|---|
| RESEARCH → POSITION | Competitive landscape mapped. Customer pain points validated (5+ interviews). Market sized (TAM/SAM/SOM). |
| POSITION → PLAN | Positioning statement approved by stakeholder. ICP defined. Messaging framework complete. |
| PLAN → LAUNCH | Launch plan has clear KPIs, timeline, assigned owners, and budget allocation. |
| LAUNCH → MEASURE | Launch materials deployed. Tracking and attribution in place. |
| MEASURE → ITERATE | Minimum 1 week of post-launch data collected. Early signals analyzed. |

## RESEARCH detail

1. **Market sizing** — TAM, SAM, SOM with methodology documented
2. **Competitive analysis** — feature matrix, positioning map, SWOT per competitor
3. **Customer interviews** — structured interview guide, 5+ interviews, synthesis
4. **Industry trends** — macro trends, regulatory changes, technology shifts

## POSITION detail

1. **ICP definition** — firmographics, technographics, pain points, buying triggers
2. **Value proposition** — what you do, for whom, how it's different, why it matters
3. **Messaging framework** — elevator pitch → short pitch → detailed pitch
4. **Differentiation** — clear articulation of "why us, not them"

## PLAN detail

1. **Channel strategy** — which channels, why, budget per channel
2. **Timeline** — pre-launch, launch, post-launch phases with milestones
3. **Owner assignments** — RACI for every launch workstream
4. **Launch checklist** — day-of tasks, monitoring, escalation paths
